# dma-controller

Two-channel AXI4 DMA with AXI-Stream bridging. The CSR block is **generated by
[reggen](../reggen/)** from [specs/dma_csr.yaml](specs/dma_csr.yaml) — the YAML
is the single source of truth for RTL, UVM model, C header, and docs.

```
              ┌──────────────────────────── dma_top ────────────────────────────┐
 AXI4-Lite ───▶ csr.sv (GENERATED)                                              │
 (from CPU)   │   per-channel CTRL/STATUS/ADDR/LEN, INT_EN, VERSION             │
              │        │                    │                                   │
              │   dma_channel 0        dma_channel 1     ── ch0/ch1 m_axis ──▶  │
              │   rd FSM ─ FIFO ─ wr   rd FSM ─ FIFO ─ wr ◀─ ch0/ch1 s_axis ──  │
              │        ╲              ╱                                         │
              │       dma_axi_mux (round-robin, burst-locked)                   │──▶ AXI4 master
              └─────────────────────────────────────────────────────────────────┘
```

## v2 features

- **Two independent channels** sharing one AXI4 master through a round-robin
  arbiter (read and write paths arbitrated separately, locked per burst).
- **Pipelined datapath**: each channel's read engine fetches ahead while its
  write engine drains — the FIFO decouples them. Space/data are reserved
  before a burst is committed, so overflow and mid-burst WVALID stalls are
  impossible by construction.
- **Three modes per channel** (`CHn_CTRL.MODE`): MM2MM copy, MM2S
  (memory → AXI-Stream, with TLAST/TKEEP), S2MM (AXI-Stream → memory).
- **Byte-granular LENGTH**: the final beat carries partial WSTRB/TKEEP.
  Addresses stay word-aligned (bits 1:0 ignored).
- Bursts never cross a 4KB boundary on either address; max 255 beats
  (`CHn_CTRL.BURST_LEN`).
- Race-free completion: DONE/ERROR are guaranteed latched before BUSY
  deasserts — software polling can never observe a completed transfer with no
  status flag.
- On a non-OKAY response the transfer aborts cleanly (`ERROR` set, `DONE`
  clear); the channel is immediately reusable.

## Programming model (per channel, base = 0x00 / 0x20)

1. Write `CHn_SRC_ADDR`, `CHn_DST_ADDR`, `CHn_LENGTH`.
2. Write `CHn_CTRL`: `MODE`, `BURST_LEN`, `START=1` (self-clearing).
3. Poll `CHn_STATUS.BUSY` or take the irq
   (`irq = |(CHn_STATUS.{DONE,ERROR} & INT_EN.CHn_*_EN)`); flags are W1C.

Full register reference: [docs/REGS.md](docs/REGS.md) (generated).

## Build & test

```bash
source ../reggen/.venv/bin/activate   # reggen + cocotb; Verilator on PATH
make regen     # YAML -> csr.sv, csr_reg_block.sv, csr.h, REGS.md
make lint      # verilator --lint-only -Wall
make sim       # cocotb functional suite (12 tests)
make sim WAVES=1   # same + dump.fst (view with gtkwave)
```

### Verification

- **cocotb** ([tb/cocotb/test_dma.py](tb/cocotb/test_dma.py)): 12 tests — reset
  /version, MM2MM (single word, multi-burst, byte tail with canary check, 4KB
  split with exact burst-address asserts, zero length, SLVERR abort+recovery),
  MM2S (TLAST/TKEEP), S2MM, per-channel IRQ masking, **concurrent dual-channel
  transfers through the arbiter**, back-to-back reuse. The AXI4 memory model
  applies random backpressure on every channel and checks the 4KB rule per
  burst; [axi_checker.sv](tb/cocotb/axi_checker.sv) asserts VALID/payload
  stability in RTL on every run.
- **UVM RAL** ([tb/uvm/](tb/uvm/)): AXI-Lite agent + `uvm_reg_adapter` around
  the reggen-generated register model; runs `uvm_reg_hw_reset_seq` plus a
  write/read/mirror smoke sequence. Compiles clean against UVM 1.2 with
  QuestaSim `vlog`; running it requires a QuestaSim license (the bundled
  Starter license had expired at development time — renew via Intel's
  licensing portal, then:
  `vsim -c work.tb_top +UVM_TESTNAME=dma_ral_test -do "run -all; quit -f"`).

### Synthesis & timing

```bash
cd syn && quartus_sh -t synth.tcl && quartus_map dma_top \
  && quartus_fit dma_top && quartus_sta dma_top
```

Targets Cyclone V (5CGXFC7C7F23C8) at a 100 MHz constraint with virtual pins
(IP characterization). Measured (Quartus Lite 21.1, slow 1100mV 85C model):

| Metric | Result |
|---|---|
| Logic | 1,127 ALMs (2%), 958 registers |
| Memory | 2 M10K blocks (the two 256×32 channel FIFOs) |
| Core reg-to-reg timing | **met at 100 MHz, worst slack +1.77 ns** |
| I/O-inclusive Fmax | 73.9 MHz (virtual-pin budget dominated; register the AXI outputs at integration) |

The original single-cycle burst-size computation (31-bit clamp + cascaded
mins) was the critical path at −3.5 ns; it is now pipelined across the
SETUP/CALC FSM states (2 idle cycles per burst), which closed core timing.

## Deferred (v3 candidates)

Scatter-gather / descriptor rings, >2 channels (arbiter generalizes), arbitrary
byte alignment (data rotation), multiple outstanding AR/AW, clock-domain
crossing variants.
